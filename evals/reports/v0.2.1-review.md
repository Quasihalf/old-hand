## Failure Baseline

Two pre-fix Codex tasks showed why `allow_implicit_invocation: true` was not
enough:

- Task `019f5c33-421e-78b2-8057-e3278d58d15d` read `old-hand` together with
  creator skills, but did not read the research protocol or call a search tool.
  It moved directly to a multiple-choice design question.
- Task `019f5c3c-4846-7412-9e2f-08b688156225` initially loaded creator and
  brainstorming skills without loading `old-hand`; no search occurred in the
  initial creation route.

Both tasks used Codex CLI 0.144.2 and `gpt-5.6-sol`. These are qualitative
failure traces, not a measured invocation rate.

## Fix

Version 0.2.1 adds three portable layers without a global `AGENTS.md` rule:

1. Discovery metadata explicitly names creating or substantially redesigning
   skills, plugins, agents, MCP servers, automations, workflows, and developer
   tools, including the stage before initial design decisions are finalized.
2. The routed research protocol makes comparable-project research a creation
   gate even when the user did not ask to search. A provisional query hypothesis
   is allowed, but design, scaffolding, and final direction wait for evidence.
3. Deterministic trigger and behavior fixtures plus validator regression tests
   protect the discovery terms and research gate.

Mechanical renames, formatting, and metadata-only edits to existing artifacts
remain negative triggers.

## Evidence Boundaries

- This is one final forward run plus two qualitative baseline traces, not a
  randomized or repeated benchmark.
- It does not establish deterministic invocation, an invocation percentage,
  cross-model behavior, or cross-platform behavior.
- Trigger fixtures validate repository intent and regression coverage; they are
  not runtime telemetry.
- A Skill can strongly improve discovery and behavior after selection, but the
  host model remains the final router.
- No external project code was installed, executed, or copied during the
  acceptance run.
