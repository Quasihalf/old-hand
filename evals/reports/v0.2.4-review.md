# old-hand v0.2.4 Visible Research Brief Review

Review date: 2026-07-18 (Asia/Shanghai)
Forward-test environment: Codex CLI 0.144.1 using the GPT-5.6 family

## Purpose

This release makes research synthesis a user-visible output contract. The
previous `v0.2.3` protocol asked for borrowed patterns and a minimal direction,
but a baseline video-Skill run showed that useful decisions could still be
buried in a candidate table or omitted as an explicit common-pattern synthesis.
The change does not ask Codex to reveal hidden reasoning. It requires a compact,
auditable decision brief containing the conclusions and their evidence.

## Contract

After project-start external research and before design, scaffolding, or
implementation, the response must show four separate sections:

1. **Common Mature Patterns:** 3-5 patterns synthesized across candidates.
2. **Borrow / Avoid:** concrete adoption, rejection, or deferral decisions with
   fit, complexity, maintenance, or license reasons.
3. **Minimal Build Direction:** the smallest direction supported by the evidence.
4. **Decision-Critical Questions:** only questions that can change the direction,
   or `None` when no blocker remains.

The candidate table remains a source-evidence section and cannot replace these
four synthesis sections. An execution request shows the brief and continues
without waiting unless one of the questions blocks a correct implementation.

## Baseline Failure

The installed `v0.2.3` baseline was run with the natural prompt “帮我做一个剪辑视频的
Skill。现在只完成前期调研和方向建议，不要创建或修改任何文件。” It searched
successfully and produced useful recommendations, but the synthesis was not
stable: common patterns were embedded in prose, and borrow/avoid decisions were
mixed into the candidate comparison rather than displayed as a separate section.
The baseline artifact is retained locally at `/tmp/old-hand-v024-baseline/final.md`.

## Candidate Forward Tests

Three independent prompts were run against the candidate `skills/old-hand`
directory in isolated working directories. The prompts did not name the four
required headings or describe the expected answer shape.

| Scenario | External search | Common patterns | Borrow / Avoid | Minimal direction | Decision questions | Files changed |
| --- | ---: | ---: | ---: | ---: | ---: | ---: |
| New video-editing Skill | yes | yes | yes | yes | yes | no |
| Local-first macOS clipboard product | yes | yes | yes | yes | yes | no |
| Offline-first embedded database selection | yes | yes | yes | yes | yes | no |

The raw traces and final responses are retained locally under
`/tmp/old-hand-v024-forward-final/{video,clipboard,database}`. Independent
inspection confirmed that each trace read the candidate Old Hand and research
protocol, completed an external search, and emitted all four sections in the
final response. The video run selected the existing local `edit-video` Skill as
the baseline rather than scaffolding a duplicate; the clipboard run converged
on a native bounded-history design; the database run separated SQLite’s
transactional core from optional sync, analytics, and CRDT concerns.

## Installed-State Smoke

The installed `v0.2.4` plugin was then tested in a fresh ephemeral Codex run
with the natural prompt “帮我做一个剪辑视频的 Skill。现在先完成前期调研并给出方向建议，
不要创建或修改任何文件。” The prompt did not name Old Hand or prescribe the
four-section response shape.

The trace shows that Codex:

- selected Old Hand without an explicit invocation;
- read the installed `SKILL.md` and `references/research-protocol.md` from the
  `v0.2.4` plugin cache;
- used successful `gh search repos`, `gh search code`, `gh repo view`, and
  `gh api` calls to inspect more than eight external candidates;
- deep-read the closest relevant projects and compared them with the installed
  local `edit-video` Skill;
- emitted separate `Common Mature Patterns`, `Borrow / Avoid`, `Minimal Build
  Direction`, and `Decision-Critical Questions` sections; and
- created no requested project or Skill files.

The first direct cache read used an incomplete legacy path and failed; Codex
located the versioned cache path and continued successfully. The CLI also
emitted unrelated 401 warnings while reconnecting Apps and analytics services,
but the run completed with exit code 0 and the GitHub research calls succeeded.
The raw response and trace are retained locally at
`/tmp/old-hand-v024-installed-visible`.

## Boundaries

- The matrix measures one model family, one machine, one Codex version, and one
  search configuration; it is not a universal output guarantee.
- Research remains limited to project-start directions and material foundational
  choices. Ordinary local implementation and small debugging stay local-first.
- The brief exposes decisions and evidence, not private chain-of-thought.
