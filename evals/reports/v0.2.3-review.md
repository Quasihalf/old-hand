# old-hand v0.2.3 Substantive Coding Routing Review

Review date: 2026-07-18 (Asia/Shanghai)
Forward-test environment: Codex CLI 0.144.1 using the GPT-5.6 family

## Result: PASS

Version `0.2.3` broadens implicit routing from creator-heavy discovery to every
substantive coding task while preserving two boundaries: ordinary work in an
existing project does not browse by default, and purely mechanical edits do not
load Old Hand.

The final candidate passed 24 isolated forward runs:

| Route | Runs | Required old-hand behavior | Functional result | Search |
| --- | ---: | --- | ---: | ---: |
| Existing-project implementation | 9 | loaded in 9/9 | 9/9 | 0/9 |
| Local debugging | 3 | loaded and read debug protocol in 3/3 | 3/3 | 0/3 |
| Mechanical controls | 6 | skipped in 6/6 | 6/6 | 0/6 |
| New artifact or product creation | 6 | loaded, read research protocol, searched in 6/6 | gate check 6/6 | 6/6 |

## Failure Baseline

The same three implementation prompts were first run against installed
`v0.2.2`, three times each. Only 7/9 traces actually read
`skills/old-hand/SKILL.md`, although all nine implementations passed their
behavior checks and none browsed. The misses were one CLI JSON-output task and
one environment-configuration task.

That failure showed why `Normal implementation` in the Skill body was
insufficient: body instructions apply only after metadata routing has already
selected the Skill. The host `AGENTS.md` rule emphasized creator work, so the
portable Skill description needed to make ordinary substantive coding a
first-class discovery route.

## Fix

The release makes five related changes:

1. The Skill description begins with every substantive coding task and names
   feature implementation in existing code, debugging, refactoring, review,
   dependency choice, and architecture decisions.
2. It explicitly stays active beside specialized coding skills even when the
   user does not name Old Hand.
3. The mechanical boundary is also front-loaded: constrained renames,
   formatting-only changes, exact metadata corrections, and one-step factual
   reads are excluded.
4. Plugin and OpenAI agent default prompts now cover substantive coding rather
   than only creator workflows.
5. Validators and public eval fixtures lock the broader positive routes and the
   narrow negative boundary.

## Environment Boundaries

- Each run used a fresh temporary workspace and Codex home with the candidate
  Skill copied from the repository.
- The current compiled host `AGENTS.md` was intentionally retained. Its Old
  Hand route still emphasizes creator work, so the 9/9 implementation result
  exercises the stronger portable Skill metadata rather than a newly edited
  host rule.
- All runs used one model family, machine, Codex version, and configured search
  setup. The result is a regression matrix, not proof of universal invocation
  across every wording, language, model, or platform.
- External search results were treated as untrusted reference material. No
  downloaded project code was executed, installed, or copied.

## Release Recommendation

The verified `v0.2.3` payload satisfies the release conditions. Publish this
report-only closure commit and tag it as `v0.2.3`; no routing, protocol,
manifest, validator, or eval input changed after the installed-state smoke.
